# Fedora

> Distro id: `fedora` · Package manager: DNF · Package search: DNF metadata from `fedora` and `updates`.

## What this page covers

This page documents the `fedora` backend: supported releases, build assumptions, command examples, generated environment variables, and Fedora-specific troubleshooting.

## When to choose it

Choose Fedora when you want a modern DNF-based desktop with recent kernels and desktop stacks.

## Supported branches / releases

| Branch / release |
| --- |
| `stable` |
| `latest` |
| `rawhide` |
| `42` |
| `41` |

Defaults:

- Branch/release: `stable`
- Architecture: `x86_64`; choices: `x86_64`
- User: `fedora`
- Hostname: `ledit-fedora`
- Output image name: `ledit-fedora.img`

## Backend implementation

| Role | Path |
| --- | --- |
| Build backend | `backend/scripts/build-fedora-usb.sh` |
| Configure backend | `none; plan is generated by Python and consumed by the build script` |
| Branch environment variable | `FEDORA_RELEASE` |
| Distro environment prefix | `FEDORA_USB` |
| Shared profile prefix | `LEDIT_USB` |

Fedora uses a Python planning step before the shell builder runs. The planner computes packages, groups, services, default target, display manager, session, and warnings from the selected LEDIT profile.

## Host requirements

Native Linux is the safest path for full image creation. Install `dnf`, GRUB2 EFI tools, `parted`, `dosfstools`, and `e2fsprogs`. macOS can still use search and dry-run, but validate the full builder path before relying on it.

## Quick commands

```sh
./ledit distros
./ledit search firefox --distro fedora --branch stable --limit 5
./ledit build --distro fedora --branch stable --dry-run --ask-password -y
./ledit build --distro fedora --branch stable --ask-password -y

./ledit build --distro fedora \
  --branch stable \
  --desktop gnome \
  --image-size 32G \
  --output "$HOME/Downloads/ledit-fedora.img" \
  --ask-password \
  --extra-packages "vim htop podman" \
  -y
```

## GUI and TUI notes

In the GUI, select **Fedora**. The release selector exposes `stable`, `latest`, `rawhide`, and numeric releases. Package suggestions use Fedora/DNF metadata.

In the TUI, run `./ledit tui` and select `fedora`.

## Build profiles

| Profile | Use it when |
| --- | --- |
| `compatibility` | You want Fedora with a broad desktop/hardware default. |
| `minimal` | You want to start from fewer packages and explicitly add what is needed. |

Example:

```sh
./ledit build --distro fedora --profile minimal --desktop none --browser none --ask-password -y
```

## Bootloader support

GRUB and systemd-boot are accepted by the profile planner. extlinux is not supported.

## Environment variable reference

The real prefix for this backend is `FEDORA_USB`. The backend also receives `LEDIT_USB_*` where appropriate.

| Variable family | Meaning |
| --- | --- |
| `FEDORA_RELEASE` | Selected release/branch. |
| `FEDORA_USB_PACKAGES` | Rendered package list generated by the planner. |
| `FEDORA_USB_GROUPS` | DNF group list generated by the planner. |
| `FEDORA_USB_SERVICES` | Services to enable in the target image. |
| `FEDORA_USB_DEFAULT_TARGET` | systemd default target. |
| `FEDORA_USB_WARNINGS` | Planner warnings shown during dry-run/build. |
| `*_USER`, `*_HOSTNAME` | Initial identity. |
| `*_DESKTOP`, `*_DISPLAY_MANAGER`, `*_DEFAULT_SESSION`, `*_TILING_WMS` | Desktop/session choices. |
| `*_NETWORK`, `*_WIFI`, `*_BLUETOOTH`, `*_AUDIO`, `*_BROWSER` | Services and desktop integration. |
| `*_BOOTLOADER`, `*_KERNEL_FLAVOR`, `*_FIRMWARE`, `*_AUTO_RESIZE` | Boot and hardware settings. |
| `*_EXTRA_PACKAGES` | Extra DNF package names. |

```txt
FEDORA_RELEASE=stable
FEDORA_USB_DRY_RUN=1
LEDIT_USB_DRY_RUN=1
```

## Troubleshooting

| Symptom | What to check |
| --- | --- |
| Dry-run prints warnings | Read `FEDORA_USB_WARNINGS`; the planner is telling you what was adjusted. |
| Package not found | Search with the exact Fedora release: `./ledit search <pkg> --distro fedora --branch 42`. |
| Build fails on macOS | Prefer Linux for full Fedora image creation until your Docker path is validated. |
| Image does not boot | Try GRUB, keep firmware enabled, and test with the compatibility profile. |

## Backend notes

- Fedora uses a Python plan step to compute packages, groups, services, and default target.
- Dry-run renders `FEDORA_USB_PACKAGES`, `FEDORA_USB_GROUPS`, `FEDORA_USB_SERVICES`, and warnings.
- Full image output should be manually smoke-tested before flashing to production USB devices.
