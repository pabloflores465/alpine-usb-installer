# RHEL family (Rocky/Alma/CentOS Stream compatible)

> Distro id: `rhel` · Package manager: DNF · Package search: RHEL-family metadata from `baseos` and `appstream`.

## What this page covers

This page documents the RHEL-family backend used for Rocky Linux, AlmaLinux, and CentOS Stream compatible flows.

## When to choose it

Choose this backend when you want a Rocky/Alma/CentOS Stream compatible image with DNF package planning.

## Supported branches / releases

| Branch / release |
| --- |
| `9` |
| `10` |

Defaults:

- Branch/release: `9`
- Architecture: `x86_64`; choices: `x86_64`
- User: `linux`
- Hostname: `ledit-rhel`
- Output image name: `ledit-rhel.img`

## Backend implementation

| Role | Path |
| --- | --- |
| Build backend | `backend/scripts/build-rhel-usb.sh` |
| Configure backend | `backend/scripts/configure-rhel-usb.sh` |
| Branch environment variable | `RHEL_USB_RELEASE` |
| Distro environment prefix | `RHEL_USB` |
| Shared profile prefix | `LEDIT_USB` |

Aliases `rocky`, `rockylinux`, `alma`, `almalinux`, `centos`, and `centos-stream` resolve to this backend. The variant is passed through `RHEL_USB_DISTRO`.

## Host requirements

Docker Desktop on macOS using a Rocky-compatible container path. Native Linux requires DNF installroot support, GRUB2 EFI tools, `parted`, `dosfstools`, and `e2fsprogs`.

## Quick commands

```sh
./ledit distros
./ledit search firefox --distro rhel --branch 9 --limit 5
./ledit build --distro rhel --branch 9 --dry-run --ask-password -y
./ledit build --distro rhel --branch 9 --ask-password -y

# Alias examples.
./ledit build --distro rocky --branch 9 --ask-password -y
./ledit build --distro alma --branch 9 --ask-password -y
./ledit build --distro centos-stream --branch 9 --ask-password -y

./ledit build --distro rhel \
  --branch 9 \
  --image-size 24G \
  --output "$HOME/Downloads/ledit-rhel.img" \
  --ask-password \
  --extra-packages "vim htop podman" \
  -y
```

## GUI and TUI notes

In the GUI, select **RHEL family**. Use aliases from the CLI when you need a specific compatible variant.

In the TUI, run `./ledit tui` and select `rhel`.

## Build profiles

| Profile | Use it when |
| --- | --- |
| `compatibility` | You want a desktop/hardware-friendly DNF image. |
| `minimal` | You want fewer packages and will add services manually. |

## Bootloader support

GRUB and systemd-boot are supported by the profile layer. extlinux is not supported.

## Environment variable reference

The real prefix for this backend is `RHEL_USB`. The backend also receives the shared `LEDIT_USB_*` variables.

| Variable family | Meaning |
| --- | --- |
| `RHEL_USB_RELEASE` | Major release, for example `9` or `10`. |
| `RHEL_USB_DISTRO` | Variant selector: Rocky, Alma, or CentOS Stream compatible. |
| `RHEL_USB_PACKAGE_LIST` | Resolved package list generated by LEDIT. |
| `*_PROFILE` | `compatibility` or `minimal` build preset. |
| `*_USER`, `*_HOSTNAME` | Initial identity. |
| `*_PASSWORD_FILE`, `*_ROOT_PASSWORD_FILE` | Temporary secret files. |
| `*_DESKTOP`, `*_DISPLAY_MANAGER`, `*_DEFAULT_SESSION`, `*_TILING_WMS` | Desktop/session choices. |
| `*_NETWORK`, `*_WIFI`, `*_BLUETOOTH`, `*_AUDIO`, `*_BROWSER` | Services and desktop integration. |
| `*_BOOTLOADER`, `*_KERNEL_FLAVOR`, `*_FIRMWARE`, `*_AUTO_RESIZE` | Boot and hardware settings. |
| `*_EXTRA_PACKAGES` | Extra DNF package names. |

```txt
RHEL_USB_RELEASE=9
RHEL_USB_DISTRO=rocky
RHEL_USB_DRY_RUN=1
LEDIT_USB_DRY_RUN=1
```

## Troubleshooting

| Symptom | What to check |
| --- | --- |
| Alias behaves unexpectedly | Use canonical `--distro rhel`, or verify the alias maps to the intended variant. |
| Package not found | Search the selected major release and verify BaseOS/AppStream availability. |
| Build fails in Docker | Confirm Docker is running and has enough disk space for DNF installroot work. |
| Image does not boot | Try GRUB first and keep firmware enabled. |

## Backend notes

- Aliases `rocky`, `rockylinux`, `alma`, `almalinux`, `centos`, and `centos-stream` resolve to this backend.
- `RHEL_USB_DISTRO` controls the variant used by the builder.
- The package list is resolved by `ledit_core/rhel_packages/packages.py`.
